# Contributions individuelles

> À compléter par l'équipe. Le cahier pédagogique exige des « contributions
> individuelles détaillées » et une prise de parole individuelle en vidéo.
> Modèle ci-dessous (équipe de 2 à 4 membres).

## Répartition des rôles (proposition)

| Membre | Rôle principal | Domaines |
|---|---|---|
| _Nom Prénom 1_ | Lead Dev / Backend | API FastAPI, modèle de données, tests |
| _Nom Prénom 2_ | DevOps / Infra | Terraform, AKS, Helm, CI/CD |
| _Nom Prénom 3_ | SRE / Observabilité | Monitoring, alerting, PRA/PCA |
| _Nom Prénom 4_ | DevSecOps / FinOps | Sécurité, scans, gestion des coûts, doc |

> En équipe réduite (2 personnes), regrouper les rôles deux à deux.

## Détail des contributions (à remplir)

### _Nom Prénom 1_
- Réalisations : …
- Difficultés rencontrées : …
- Compétences développées : …
- Axes d'amélioration : …

### _Nom Prénom 2_
- Réalisations : …
- …

*(Reproduire pour chaque membre. Cette section alimente le rendu individuel
PDF : « réflexion personnelle, forces/faiblesses, compétences développées ».)*

## Traçabilité

La contribution réelle de chacun est objectivable via :
- l'historique Git (`git shortlog -sn`),
- les Pull Requests et revues de code,
- le tableau de bord des contributeurs GitHub (Insights → Contributors).
